# Lab Storage Manager — Agent Instructions

**ALWAYS READ FIRST:** [CLAUDE.md](./CLAUDE.md) for project identity, stack, constraints, and folder structure.

## Critical Context

This is a **warehouse management system** (Lab Storage Manager) for electronics testing labs and R&D centers (Visteon).

**Tech Stack (LOCKED — Do not change):**
- Frontend: React 19 + Vite + TypeScript
- Backend: Node.js + Fastify + TypeScript
- ORM: Prisma | Database: PostgreSQL
- Auth: LDAP → JWT | Reverse proxy: Nginx
- Barcodes: Code 128 (1D) + QR Code (2D)
- CSS: Tailwind v3 (NOT v4)
- TypeScript: `erasableSyntaxOnly: true` — **NO enums, use const + type pattern**

## Roles & Permissions

| Role | Access |
|------|--------|
| Admin | Everything: config, user mgmt, edit scrapped items, location hierarchy |
| User | Warehouse ops: receipt, move, exit, return, scrap, consume, print labels |
| Viewer | Read-only: queries + reports |

- Auth: LDAP → JWT with role claim
- Users never hard-deleted, only `isActive: false`

## Item Types (6 Total)

1. **Electronics Sample** — OEM part, tracked individually, 11 fields
2. **Fixture (Mechanical)** — vibration/shock fixtures, multi-select type
3. **Fixture (Climatic)** — climatic/dust/salt/water fixtures
4. **Spare Part** — machine parts with manufacturer/model/type
5. **Consumable** — quantity-tracked, expiry date, lot number
6. **Misc** — non-specific, fewer attributes

## Storage Hierarchy

```
Site (e.g. Sofia, Munich, Paris)
└── Building (Main, Lab…)
    └── Storage Area (A, B, C…)
        └── Location: [Area]-[Row]-[Shelf]-[Level]  e.g. A-01-02-5
```

- Site + Building form namespace — same area code can exist in multiple buildings/sites
- External Locations — external labs with contact person, address, city (temporary exits)

## Warehouse Operations

| Op | What it does |
|----|---|
| Receipt | Item enters storage; date, person, facility recorded |
| Move | Between containers, shelves, areas, or sites |
| Exit (Temp) | Item leaves to external location; expected return date tracked |
| Return | Scan-in required; overdue items flagged |
| Scrap | Item marked scrapped; becomes read-only in UI (admin-only edits) |
| Consume | Reduce quantity of consumable; records quantity + person |
